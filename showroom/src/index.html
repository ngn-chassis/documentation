Typography Layout UI Components

Chassis is a CSS pre-processor and UI framework for modern responsive UI development.

Chassis consists of:

  • PostCSS-based pre-processor allows for the use of CSS4 syntax today
  • Typography-based layout engine automatically scales every element of your UI to the ideal dimensions for the viewport
  • A suite of modular, themeable and extendable web components
  • Convenient Mixins and Functions to speed development
  • A bunch of cool extras to make UI development painless

Typography

Mixins

font-size

@chassis font-size [alias] [mult(int)];

Use the font-size mixin to apply a calculated font size to any element. If you would like to set a font-size between any two of the pre-calculated values, you can use the mult() function. It accepts any decimal number and will multiply the resulting calculated font-size against this decimal.

ellipsis

@chassis ellipsis;

Apply an ellipsis to the targeted element.

NOTE Ellipsis does not work on inline elements.

Layout

Mixins

constrain-width

@chassis constrain-width [min(int)] [max(int)] [gutter()];

Constrain the width of an element to min and max values. gutter will apply padding to the left and right sides of the element. By default, constrain-width uses your min and max layout width values as well as your layout gutter. This allows you to create full-width container elements and constrain the width of the content inside.

Helper class .width constraint applies this mixin with the default values.

inline-block-layout

@chassis inline-block-layout [no-margin[-right[-bottom]]] [no-padding[-top[-right[-bottom[-left]]]]] [no-line-height] [multi-line];

Apply calculated margin, padding and line-height matching Chassis' built-in inline-block elements, like buttons and form fields.

z-index

@chassis z-index [value or component];

Chassis include the following default z-index values:

min -1000
behind -1
default 0
front 1
max 1000

It also maintains a stack of components. You can insert components into this stack to have their default z-index automatically calculated. This relieves you of having to manage z-index manually.

UI Components

Chassis UI components are totally modular and completely themeable. They are built to work seamlessly with Chassis' Auto-Typography engine, so they are always proportioned correctly based on your core typography rules. Chassis provides a series of mixins for working with components:

Built-in UI Components

Anchor

Chassis links fully reset the default browser styles and are themeable.

<a href="http://github.com/ngnjs">Default Anchor (Link)</a>
Default Anchor (Link)

If anchor-button is also included, adding class "button" to an anchor tag will apply anchor-button styles to it. Chassis is smart enough to know which properties need to be persisted from the default anchor, while resetting any properties that might conflict.

<a class="button" href="http://github.com/ngnjs">Anchor Button</a>
Anchor Button

Button

Chassis' button component has calculated margin, padding and line-height values by default. Its proportions are determined by your base typography rules.

Default
<button>Default Button</button>
Pill

Any button can be made into a pill shape simply by applying class "pill".

<button class="pill">Pill Button</a>

Tag

Tags display inline with text and can have icons placed in them. They come in regular and pill-shaped varieties.

Mixins

include

@chassis include [component [more components...]];

To avoid unused CSS, Chassis will not load a UI component's styles unless it is first included. Components can have additional styles passed in through a theme file, or at the point of inclusion via CSS declarations. See extend for more information.

This mixin also accepts all as a value, in which case all Chassis UI Components will be loaded.

new

@chassis new [component] {...};

Creates a new, unthemed instance of the chosen component. Accepts css or theme rules.

new uses a special syntax which is the same as that used in theme files.

extend

@chassis extend [component] {...};

Chassis components can be extended with custom properties. This provides a way to create different versions of components on-the-fly.

extend uses a special syntax which is the same as that used in theme files.
